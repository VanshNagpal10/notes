1. Files and File Paths

UNIT III · Fundamental Concept
Definition

A file is a named location on disk that stores information permanently. A file path is the address that tells the computer exactly where a file is stored — like a street address for your data.

Types of File Paths

🗺️ Absolute vs Relative Paths
ABSOLUTE PATH — starts from root, complete address
──────────────────────────────────────────────────
Windows :  C:\Users\Vansh\Documents\project\main.py
Mac/Linux: /home/vansh/Documents/project/main.py

RELATIVE PATH — relative to the CURRENT WORKING DIRECTORY
──────────────────────────────────────────────────
Current Dir: C:\Users\Vansh\Documents\
Relative:    project\main.py      ← goes INTO project folder
             .\main.py           ← dot (.) = current folder
             ..\other.py         ← dotdot (..) = go UP one level

Path Separators

OSSeparatorExample
Windows\ (backslash)C:\Users\Vansh\file.txt
Mac / Linux/ (forward slash)/home/vansh/file.txt
⚠️ Windows Tip: In Python strings, \ is an escape character. Use raw strings r"C:\Users\file" or double backslash "C:\\Users\\file" to avoid errors.
 file_paths.py
import os

# Get current working directory
cwd = os.getcwd()
print(cwd)  # e.g., C:\Users\Vansh

# Change the working directory
os.chdir('C:\\Users\\Vansh\\Documents')

# Join paths safely (works on ALL operating systems)
path = os.path.join('Users', 'Vansh', 'notes.txt')
print(path)  # Users\Vansh\notes.txt (Windows) or Users/Vansh/notes.txt (Linux)

# List all files and folders in current directory
items = os.listdir('.')
print(items)  # ['file1.txt', 'folder1', 'script.py', ...]
✅ Best Practice: Always use os.path.join() instead of manually typing / or \. This makes your code work on Windows AND Mac/Linux!

Checking if a Path Exists

 check_path.py
import os

# Does the path exist at all?
print(os.path.exists('C:\\Windows'))     # True
print(os.path.exists('C:\\FakeFolder'))  # False

# Is it a file or a folder?
print(os.path.isfile('notes.txt'))    # True if it's a file
print(os.path.isdir('Documents'))    # True if it's a folder

import os

Method 1: Traditional open/close

 traditional_file.py
# ─── WRITING to a file ───────────────────────────────────
file = open('hello.txt', 'w')  # open for writing (creates/overwrites)
file.write('Hello, World!\n')   # \n = new line
file.write('This is line 2.\n')
file.close()  # ALWAYS close the file!

# ─── READING from a file ─────────────────────────────────
file = open('hello.txt', 'r')  # open for reading
content = file.read()         # read ALL content as one string
print(content)
file.close()

# ─── READING LINE BY LINE ───────────────────────────────
file = open('hello.txt', 'r')
lines = file.readlines()  # returns a LIST of lines
print(lines)
# ['Hello, World!\n', 'This is line 2.\n']
file.close()

Method 2: Using with Statement (RECOMMENDED ⭐)

 with_statement.py
# The 'with' block AUTOMATICALLY closes the file — even if errors occur!

# Writing
with open('diary.txt', 'w') as f:
    f.write('Day 1: Started studying Python\n')
    f.write('Day 2: Learned file handling\n')
# file is automatically closed here ✅

# Reading
with open('diary.txt', 'r') as f:
    for line in f:              # iterate line by line (memory efficient!)
        print(line.strip())    # strip() removes trailing \n

# Appending (add to end without deleting existing content)
with open('diary.txt', 'a') as f:
    f.write('Day 3: Practiced os.path module\n')

# Reading all lines into a list
with open('diary.txt', 'r') as f:
    all_lines = f.readlines()  # ['Day 1: ...\n', 'Day 2: ...\n']
    single_line = f.readline() # reads just ONE line
📌 Key Difference: read() returns one big string. readlines() returns a list where each item is a line. readline() reads just one line at a time (useful for huge files).

4. Saving Variables with the shelve Module

UNIT III · Persistent Data Storage
Definition

The shelve module lets you save Python variables (like lists, dicts, objects) to a file and load them back later. It works like a persistent dictionary — you store values using keys, just like a regular dict, but the data survives even when the program exits.

🗄️ How shelve Works
 Python Program            shelve.open('mydata')         Disk Files Created
 ──────────────           ─────────────────────          ──────────────────
 cats = ['Zoe',    ───►   shelf['cats'] = cats    ───►   mydata.bak
         'Luna']                                          mydata.dat
                                                          mydata.dir

 Later (even after restart):

 shelf = shelve          shelve.open('mydata')         
   .open('mydata') ◄───  shelf['cats']            ◄───   mydata.dat
 print(shelf['cats'])
 ['Zoe', 'Luna']  ✅  Data is preserved!
 shelve_demo.py
import shelve

# ── SAVING data ──────────────────────────────────────────
with shelve.open('myData') as shelf:
    shelf['cats'] = ['Zoe', 'Luna', 'Mochi']
    shelf['age']  = 21
    shelf['scores'] = {'math': 95, 'python': 88}
# ← file automatically closed, data is saved to disk

# ── LOADING data ─────────────────────────────────────────
with shelve.open('myData') as shelf:
    print(shelf['cats'])     # ['Zoe', 'Luna', 'Mochi']
    print(shelf['age'])      # 21
    print(shelf['scores'])   # {'math': 95, 'python': 88}

# ── KEYS and VALUES (like a regular dict!) ────────────────
with shelve.open('myData') as shelf:
    print(list(shelf.keys()))    # ['cats', 'age', 'scores']
    print(list(shelf.values())) # all stored values

# ── DELETING an entry ─────────────────────────────────────
with shelve.open('myData') as shelf:
    del shelf['age']  # removes the 'age' key
📌 shelve vs regular files: Regular files only store text. shelve stores actual Python objects — lists, dicts, numbers, even custom objects — without converting them to text first!
✅ Files created by shelve: On Windows, shelve.open('myData') creates 3 files: myData.bak, myData.dat, myData.dir. On Mac/Linux, it might just create one file. Don't worry about these files — Python manages them automatically.

5. Saving Variables with pprint.pformat()

UNIT III · Pretty Printing & Variable Persistence
Definition

pprint stands for Pretty Print. The pprint.pformat() function converts a Python data structure (like a list or dict) into a nicely formatted, readable string that can be saved to a .py file and imported back later.

pprint vs print

 pprint_comparison.py
import pprint

data = {'name': 'Vansh', 'marks': [88, 92, 79, 95], 'city': 'Delhi', 'year': 2}

# Regular print — everything on ONE line, hard to read
print(data)
# {'name': 'Vansh', 'marks': [88, 92, 79, 95], 'city': 'Delhi', 'year': 2}

# pprint — formatted nicely, each key on its own line
pprint.pprint(data)
# {'city': 'Delhi',
#  'marks': [88, 92, 79, 95],
#  'name': 'Vansh',
#  'year': 2}

# pformat — returns the string (instead of printing it)
formatted_string = pprint.pformat(data)
print(type(formatted_string))  # <class 'str'>

Saving Variables to a .py file using pformat()

 save_with_pformat.py
import pprint

# Our data we want to save
student_data = {
    'name': 'Vansh',
    'subjects': ['Python', 'DBMS', 'OS'],
    'gpa': 8.7
}

# SAVING — write data as valid Python code
with open('student_data.py', 'w') as f:
    f.write('student_data = ' + pprint.pformat(student_data))

# student_data.py now contains:
# student_data = {'gpa': 8.7,
#                 'name': 'Vansh',
#                 'subjects': ['Python', 'DBMS', 'OS']}

# LOADING — import it like a regular Python file!
import student_data as sd
print(sd.student_data['name'])  # Vansh

6. The shutil Module

UNIT III · File & Directory Operations
Definition

shutil stands for Shell Utility. It provides high-level functions for copying, moving, renaming, and deleting files and entire directory trees — things you'd normally do with right-click in File Explorer, but done with code.

Key shutil Functions

FunctionWhat it Does
shutil.copy(src, dst)Copies a file from src to dst (just file content)
shutil.copy2(src, dst)Copies file + preserves metadata (timestamps etc.)
shutil.copytree(src, dst)Copies entire folder and all its contents
shutil.move(src, dst)Moves/renames a file or folder
shutil.rmtree(path)Deletes entire folder and all contents ⚠️ PERMANENT
 shutil_demo.py
import shutil, os

# ── COPY a single file ────────────────────────────────────
shutil.copy('notes.txt', 'backup_notes.txt')
# Creates backup_notes.txt with same content

shutil.copy('notes.txt', 'C:\\Backup\\')
# Copies notes.txt INTO the Backup folder

# ── COPY an entire folder ─────────────────────────────────
shutil.copytree('C:\\MyProject', 'C:\\MyProject_Backup')
# MyProject_Backup must NOT already exist!

# ── MOVE / RENAME ─────────────────────────────────────────
shutil.move('notes.txt', 'C:\\Documents\\notes.txt')
# Moves the file to Documents folder

shutil.move('old_name.txt', 'new_name.txt')
# Renames the file (in same directory)

# ── DELETE a folder permanently ⚠️ ───────────────────────
shutil.rmtree('C:\\TempFolder')
# Deletes TempFolder and EVERYTHING inside it — NO RECYCLE BIN!

Safe Deletion: Using send2trash (Recycle Bin)

 safe_delete.py
# Install first: pip install send2trash
import send2trash

# Safer! Sends to Recycle Bin instead of permanent deletion
send2trash.send2trash('old_notes.txt')
# Can be recovered from Recycle Bin ✅
⚠️ Warning: shutil.rmtree() permanently deletes files — they do NOT go to the Recycle Bin. Always double-check the path before calling it. Prefer send2trash for safety.

7. Walking a Directory Tree

UNIT III · Recursive Directory Traversal
Definition

"Walking" a directory tree means visiting every folder and subfolder recursively — going deeper and deeper into nested folders and listing all files found at each level. Python's os.walk() function does this automatically.

🌳 Directory Tree Example
 C:\MyFiles\                   ← os.walk() starts HERE
 ├── readme.txt
 ├── notes.py
 ├── Photos\                  ← walks INTO this folder
 │   ├── holiday.jpg
 │   └── profile.png
 └── Projects\                ← then walks INTO this folder
     ├── main.py
     └── Utils\               ← then walks INTO nested folder
         └── helper.py

 os.walk() visits: MyFiles → Photos → Projects → Utils
 Produces 4 tuples, one per folder

Understanding os.walk() Output

For each folder it visits, os.walk() gives you a tuple of 3 values:

Variable 1
dirpath

The path of the current folder being visited

Variable 2
dirnames

List of subfolder names inside current folder

Variable 3
filenames

List of file names inside current folder

 os_walk_demo.py
import os

# ── Basic walk ────────────────────────────────────────────
for dirpath, dirnames, filenames in os.walk(r'C:\MyFiles'):
    print(f'Current folder: {dirpath}')
    print(f'  Subfolders: {dirnames}')
    print(f'  Files: {filenames}')
    print()

# Output:
# Current folder: C:\MyFiles
#   Subfolders: ['Photos', 'Projects']
#   Files: ['readme.txt', 'notes.py']
#
# Current folder: C:\MyFiles\Photos
#   Subfolders: []
#   Files: ['holiday.jpg', 'profile.png']

# ── PRACTICAL: Find ALL .txt files in a directory tree ────
print("All .txt files found:")
for dirpath, dirnames, filenames in os.walk(r'C:\MyFiles'):
    for filename in filenames:
        if filename.endswith('.txt'):
            full_path = os.path.join(dirpath, filename)
            print(full_path)

# ── PRACTICAL: Calculate total size of all files ──────────
total_size = 0
for dirpath, dirnames, filenames in os.walk(r'C:\MyFiles'):
    for filename in filenames:
        full_path = os.path.join(dirpath, filename)
        total_size += os.path.getsize(full_path)
print(f'Total: {total_size} bytes')

8. Compressing Files with zipfile Module

UNIT III · File Compression
Definition

The zipfile module lets you create, read, extract, and inspect ZIP archives programmatically. A ZIP file compresses one or more files into a smaller file for storage or transfer.

🗜️ ZIP Operations Flow
   Files on Disk         zipfile Module          ZIP Archive
   ─────────────         ─────────────           ─────────────
   notes.txt    ───►
   photos.jpg   ───►  ZipFile('archive.zip','w')  ───►  archive.zip
   main.py      ───►                                     (compressed!)

   Extracting:

   archive.zip  ───►  ZipFile('archive.zip','r')  ───►  notes.txt
                      .extractall('output/')             photos.jpg
                                                         main.py

ZipFile Opening Modes

ModePurpose
'r'Read an existing ZIP file
'w'Create a new ZIP file (overwrites if exists)
'a'Append files to existing ZIP
'x'Create new ZIP, error if already exists
 zipfile_demo.py
import zipfile

# ── READING a ZIP file ─────────────────────────────────────
with zipfile.ZipFile('archive.zip', 'r') as zf:
    print(zf.namelist())         # List all files inside
    # ['notes.txt', 'photos/holiday.jpg', 'main.py']
    
    info = zf.getinfo('notes.txt')
    print(info.file_size)         # original size in bytes
    print(info.compress_size)     # compressed size

# ── EXTRACTING from a ZIP file ────────────────────────────
with zipfile.ZipFile('archive.zip', 'r') as zf:
    zf.extractall('output_folder')   # extracts ALL files
    zf.extract('notes.txt', 'output')  # extract ONLY one file

# ── CREATING a ZIP file ───────────────────────────────────
import zipfile

with zipfile.ZipFile('my_backup.zip', 'w') as zf:
    zf.write('notes.txt')          # adds notes.txt
    zf.write('main.py')            # adds main.py

# ── CREATING a ZIP from an entire folder ──────────────────
import zipfile, os

with zipfile.ZipFile('project_backup.zip', 'w') as zf:
    for dirpath, dirnames, filenames in os.walk('C:\\MyProject'):
        for filename in filenames:
            full_path = os.path.join(dirpath, filename)
            zf.write(full_path)
print('Backup complete!')
✅ Real use case: Combine os.walk() with zipfile to automatically back up an entire project folder. This is a very common and exam-favourite example!

#! python3
# mapit.py - Launches a map in the browser using an address
#            from the clipboard or command line

# Install: pip install pyperclip
import webbrowser, sys, pyperclip

# ── Get the address from command line or clipboard ────────
if len(sys.argv) > 1:
    # User ran:  python mapit.py 870 Valencia St, San Francisco
    address = ' '.join(sys.argv[1:])  # join all args into one string
else:
    # User ran:  python mapit.py   (with address already copied)
    address = pyperclip.paste()        # get from clipboard

# ── Build the Google Maps URL ─────────────────────────────
# Google Maps accepts addresses as URL-encoded query string
url = 'https://www.google.com/maps/place/' + address

# ── Open in browser ───────────────────────────────────────
webbrowser.open(url)
print(f'Opening map for: {address}')

"""
Usage:
  python mapit.py 870 Valencia St, San Francisco
  python mapit.py                     ← uses clipboard
"""

10. Downloading Files with the requests Module

UNIT IV · HTTP Requests & File Downloads
Definition

The requests module is a popular third-party library that makes it easy to send HTTP requests (GET, POST, etc.) and download content from the internet. Think of it as Python making a web request just like your browser does when you visit a website.

✅ Installation: pip install requests (not built-in, must install separately)
🌐 HTTP Request/Response Cycle
 Your Python Script              Web Server (e.g. example.com)
 ──────────────────               ─────────────────────────────

 requests.get(url)   ───────────►  Server receives GET request
                                   Server processes request
                     ◄───────────  Server sends back Response

 Response Object contains:
   ├── status_code   → 200 = OK, 404 = Not Found, 500 = Error
   ├── text          → page content as a string (HTML, JSON...)
   ├── content       → page content as raw bytes (for images/files)
   └── headers       → metadata about the response

HTTP Status Codes (Must Know!)

CodeMeaningExample
200✅ OK — SuccessPage loaded fine
301↩️ Moved PermanentlyURL has changed
400❌ Bad RequestWrong URL format
403🚫 ForbiddenNo permission
404🔍 Not FoundPage doesn't exist
500💥 Server ErrorServer-side problem
 requests_basics.py
import requests

# ── Basic GET request ─────────────────────────────────────
res = requests.get('https://automatetheboringstuff.com/files/rj.txt')

# Check if it worked
print(res.status_code)  # 200 = success

# raise_for_status() will CRASH if something went wrong
res.raise_for_status()  # raises exception if status != 200

print(type(res.text))    # <class 'str'>
print(len(res.text))     # total characters downloaded
print(res.text[:250])    # print first 250 characters

# ── Check success with status code ───────────────────────
if res.status_code == 200:
    print('Download successful!')
elif res.status_code == 404:
    print('Page not found!')

 download_and_save.py
import requests

# ── Download a TEXT file ──────────────────────────────────
url = 'https://automatetheboringstuff.com/files/rj.txt'
res = requests.get(url)
res.raise_for_status()   # crash if download failed

with open('romeo_juliet.txt', 'wb') as f:
    for chunk in res.iter_content(100000):  # 100,000 bytes per chunk
        f.write(chunk)

print('Text file saved!')

# ── Download an IMAGE ─────────────────────────────────────
img_url = 'https://example.com/photo.jpg'
res = requests.get(img_url)
res.raise_for_status()

with open('photo.jpg', 'wb') as f:
    for chunk in res.iter_content(100000):
        f.write(chunk)

print('Image saved!')

# ── Download a PDF/ZIP/any binary file ───────────────────
# Same approach — always use 'wb' mode and iter_content()
⚠️ Important: Always use 'wb' (write binary) mode when saving downloaded files — even for text files. This prevents Python from converting newline characters which could corrupt the file on Windows!
📌 .text vs .content: Use res.text to get the content as a string (for HTML, JSON, text). Use res.content for raw bytes (for images, PDFs, ZIPs). When saving to disk, always use iter_content() for efficiency.

11. HTML — HyperText Markup Language

UNIT IV · Web Page Structure & Parsing
Definition

HTML (HyperText Markup Language) is the standard language used to create web pages. It uses tags (surrounded by angle brackets like <tag>) to define the structure and content of a web page. Web scraping involves downloading HTML and then extracting information from it.

🏗️ HTML Page Structure
 <html>
 │
 ├── <head>         ← Invisible metadata
 │   ├── <title>My Page</title>      ← Browser tab title
 │   └── <meta charset="utf-8">       ← Character encoding
 │
 └── <body>         ← Everything visible to user
     ├── <h1>Heading</h1>                ← Big heading
     ├── <p>Paragraph text here</p>      ← Text paragraph
     ├── <a href="url">Link text</a>    ← Hyperlink
     ├── <img src="photo.jpg">           ← Image
     ├── <div class="box">...</div>     ← Container/box
     └── <span id="name">...</span>   ← Inline container

Parsing HTML with BeautifulSoup

Definition

BeautifulSoup is a Python library that parses (reads and understands) HTML and makes it easy to search for and extract specific elements. Combined with requests, it forms the backbone of web scraping.

 beautifulsoup_demo.py
# pip install beautifulsoup4
import requests
from bs4 import BeautifulSoup

# ── Step 1: Download the webpage ─────────────────────────
res = requests.get('https://example.com')
res.raise_for_status()

# ── Step 2: Parse the HTML ────────────────────────────────
soup = BeautifulSoup(res.text, 'html.parser')

# ── Step 3: Extract data ──────────────────────────────────

# Find the FIRST <h1> tag
heading = soup.find('h1')
print(heading.text)          # prints just the text inside <h1>

# Find ALL <a> tags (all links)
links = soup.find_all('a')
for link in links:
    print(link.get('href'))   # extract the href attribute (URL)
    print(link.text)           # link's visible text

# Find by CSS class
items = soup.find_all('div', class_='product-item')
for item in items:
    name = item.find('span', class_='name').text
    price = item.find('span', class_='price').text
    print(f'{name}: {price}')

# Find by id
header = soup.find(id='main-header')
print(header.text)

# Select using CSS selector (like in web development)
results = soup.select('div.content p')  # p inside div.content
for r in results:
    print(r.text)

Complete Web Scraping Example — Scraping Quotes

 complete_scraper.py
import requests
from bs4 import BeautifulSoup

# Step 1: Download
url = 'http://quotes.toscrape.com/'
res = requests.get(url)
res.raise_for_status()

# Step 2: Parse
soup = BeautifulSoup(res.text, 'html.parser')

# Step 3: Find all quotes
quotes = soup.find_all('div', class_='quote')

# Step 4: Extract & display data
for quote in quotes:
    text   = quote.find('span', class_='text').text
    author = quote.find('small', class_='author').text
    print(f'"{text}" — {author}')
    print('-' * 50)

# Step 5: Save to file
with open('quotes.txt', 'w', encoding='utf-8') as f:
    for quote in quotes:
        text   = quote.find('span', class_='text').text
        author = quote.find('small', class_='author').text
        f.write(f'{text} — {author}\n')

print('Scraping done! Data saved to quotes.txt')
